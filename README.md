## Architecture Philosophy

We build **AI-native systems**, not retrofitted AI features.

### Principles

- Event-driven by default  
- Decisions are first-class entities  
- Execution is automated  
- Feedback loops are mandatory  
- Memory is structured and queryable  
- Agents operate via tools, not prompts  

---

## Core System Components

- **Event Gateway** — ingest signals and triggers  
- **Decision Orchestrator** — compute decisions  
- **Policy / Contract Layer** — constraints and rules  
- **Reasoning / Optimization** — models, heuristics, ML  
- **Automation Controller** — decisions → actions  
- **Execution Gateway** — external integrations  
- **Outcome Tracking** — measure results  
- **Decision Memory** — store decisions and outcomes  

---

## Technology Stack (Typical)

- **Data Layer:** Kafka, Debezium, Iceberg, MinIO  
- **Processing:** Spark, DuckDB, SQLMesh  
- **Orchestration:** Temporal, Airflow / Kestra  
- **ML / Optimization:** XGBoost, Pyomo, OR-Tools  
- **Agents:** LLM Gateway, Tool Registry, Graph Memory  
- **APIs:** FastAPI, MCP-compatible interfaces
